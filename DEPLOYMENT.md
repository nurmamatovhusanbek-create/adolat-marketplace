# Adolat Marketplace — Deployment Guide

## Prerequisites
- A Render.com account (free or paid)
- A GitHub repository with the code pushed
- (Optional) A custom domain

---

## Quick Deploy to Render (Free Tier)

1. **Push to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Adolat v0.7.0 — ready for deployment"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/adolat.git
   git push -u origin main
   ```

2. **Create Render Blueprint**
   - Go to https://dashboard.render.com
   - Click "New" → "Blueprint"
   - Select your GitHub repository
   - Render will detect `render.yaml` and auto-configure:
     - Web service (Next.js app)
     - PostgreSQL database
   - Click "Apply"

3. **Wait for first deploy**
   - Render will build the Docker image (~3-5 min)
   - Database will be created automatically
   - Once live, your app will be at `https://adolat-marketplace.onrender.com`

4. **Initialize the database**
   - Go to your web service in Render dashboard
   - Open the "Shell" tab
   - Run:
     ```bash
     bun run db:push
     bun run scripts/seed.ts
     bun run scripts/seed-real-docs.ts
     ```

5. **Create admin user** (optional)
   - In the shell, run:
     ```bash
     bun -e "
       const {PrismaClient} = require('@prisma/client');
       const bcrypt = require('bcryptjs');
       const db = new PrismaClient();
       (async () => {
         await db.user.create({data: {email: 'admin@adolat.uz', passwordHash: await bcrypt.hash('Demo1234', 12), name: 'Admin', role: 'ADMIN', status: 'ACTIVE'}});
         console.log('Admin created');
         await db.\$disconnect();
       })();
     "
     ```

## Render Free Tier Limitations

| Resource | Free Tier | Starter ($7/mo) |
|----------|-----------|-----------------|
| RAM | 512 MB | 512 MB - 2 GB |
| CPU | 0.1 vCPU | 0.5-1 vCPU |
| Sleep after idle | Yes (15 min) | No |
| PostgreSQL storage | 1 GB | 10 GB |
| PostgreSQL expiry | **30 days** | No expiry |
| Bandwidth | 100 GB/mo | Unlimited |

**⚠️ Free tier PostgreSQL expires after 30 days — data will be lost!**
Upgrade to Starter ($7/mo) before the 30-day mark to keep your data.

## Custom Domain

1. In Render dashboard → your web service → "Settings"
2. Add custom domain (e.g., `adolat.uz`)
3. Add DNS records as instructed by Render:
   - CNAME: `adolat.uz` → `adolat-marketplace.onrender.com`
4. Update `NEXTAUTH_URL` env var to `https://adolat.uz`

---

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string (auto-set by Render) |
| `NEXTAUTH_SECRET` | Yes | Auto-generated by Render (or set manually) |
| `NEXTAUTH_URL` | Yes | Your app URL (e.g., `https://adolat.uz`) |
| `NODE_ENV` | Yes | `production` |
| `ENABLE_SIGNUP` | No | `true` to allow user registration |
| `ENABLE_ADVOCATE_SIGNUP` | No | `true` to allow advocate registration |

## GitHub Actions CI/CD

The `.github/workflows/ci-cd.yml` file will:
1. Run lint + type check on every PR
2. Build the project to verify it compiles
3. Auto-deploy to Render when code is pushed to `main`

To enable auto-deploy:
1. Go to Render dashboard → your web service → "Settings"
2. Find "Deploy Hook" URL
3. Add it as a GitHub repository secret: `RENDER_DEPLOY_HOOK_URL`

## Troubleshooting

### "OOM killed" on Render free tier
The free tier has only 512 MB RAM. If the build fails:
- Upgrade to Starter plan ($7/mo) for 2 GB RAM
- Or set `NODE_OPTIONS=--max-old-space-size=384` in build env vars

### Database connection errors
- Check that `DATABASE_URL` is set correctly
- Render's internal PostgreSQL URL format: `postgresql://user:pass@host:5432/db`
- Run `bun run db:push` to sync the schema

### Socket.io / chat not working
The chat uses HTTP polling (3-second interval), not WebSockets. This works on all
hosting platforms without special configuration.

### File uploads not working
Files are stored in `/app/storage/uploads/` inside the container. On Render free tier,
this is ephemeral — files will be lost on redeploy. For persistent file storage:
- Upgrade to Render Disk (1 GB for $1/mo)
- Or integrate with S3/Backblaze B2
